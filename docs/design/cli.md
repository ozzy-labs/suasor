# CLI

clipanion ベース。lazy import で cold start を軽く保つ（[ADR-0001](../adr/0001-typescript-bun-stack.md) / NFR-PRF-1）。

## コマンド

```bash
suasor init [--force]                  # 設定 + DB 初期化 + ネクストステップ案内（skills install は別コマンド）
suasor onboard [--connector a,b] [--skip-auth] [--skip-sync] [--write-cron] [--json]  # 対話セットアップウィザード（connector 選択 → token 格納 → auth test → config slice 追記 → 初回 sync → scheduler/MCP 雛形・[ADR-0029]）
suasor db migrate [--vec]              # projection schema 適用（idempotent）
suasor projections rebuild [--no-progress]  # event replay で projection 再構築
suasor <connector> sync [--full] [--json] [--discover | --no-discover]  # 取り込み（github / slack / ms-graph / google / box / web / local / notion / jira）。--discover/--no-discover は Slack の discovery sweep 単発トグル（ADR-0039）
suasor sync [--connector a,b] [--concurrency N] [--continue-on-error] [--full] [--json] [--no-progress]  # 有効 connector を一括取り込み（connector 間並列・one-shot・定期実行は OS スケジューラへ委譲）
suasor sync status [--json]            # connector 別の sync 鮮度（最終 sync 時刻 / 件数 / 直近の成否）を表示（ADR-0033）
suasor <connector> auth set [--token T]  # connector の資格情報を OS keychain に保存（github / ms-graph / google / box / notion / jira、省略時 stdin）
suasor <connector> auth test [--json]    # 保存済み資格情報を検証 + identity + scopes（github / ms-graph / google / box / notion / jira）
suasor github repos [--filter S] [--json] [--no-progress]  # 可視リポジトリの列挙 + 設定ブロック出力（discovery・ADR-0030）
suasor google calendars [--filter S] [--json] [--no-progress]  # 可視カレンダーの列挙 + 設定ブロック出力（discovery・ADR-0030）
suasor box folders [--root id] [--filter S] [--json] [--no-progress]  # 可視 folder のツリー列挙 + 設定ブロック出力（discovery・ADR-0030）
suasor notion databases [--filter S] [--json] [--no-progress]  # 可視 database の列挙 + 設定ブロック出力（discovery・ADR-0030）
suasor jira projects [--filter S] [--json] [--no-progress]  # 可視 project の列挙 + 設定ブロック出力（discovery・ADR-0030）
suasor connectors list [--json]        # 登録 connector の enabled / token 設定有無を一覧（introspection）
suasor doctor [--json]                 # config/DB/embedding/connector + 保守ヒントを一括ヘルスチェック（error 検出で exit 1）
suasor config show [--effective] [--json]  # 実効 config（env > file > defaults の合成値）を表示。secret は常にマスク
suasor config edit [--editor <cmd>]    # $EDITOR で config.toml を開き、保存後に schema 検証。不正なら差し戻し（rollback）
suasor validate-config [--fix]         # config.toml の構造検証（必須欠落 / invalid / dangling / typo）。--fix で安全な除去のみ自動修正
suasor export backup [--out <path>] [--format sqlite|tgz]  # store dir の整合バックアップ（VACUUM INTO・無副作用）。既定 sqlite 単一ファイル
suasor store info [--breakdown] [--json]  # store 規模（event 数 / projection 行数 / DB ファイルサイズ / vec0 件数 / FTS 規模）。--breakdown で event type 別ヒストグラム
suasor extraction status [--json]      # 文書抽出カバレッジ（extracted / stale / pending）+ backend / version
suasor extraction list-pending [--limit N] [--json]  # (re)抽出待ち source の drilldown（pending / stale）
suasor embeddings status [--json]      # 埋め込みカバレッジ（entity 種別ごとの embedded / pending / stale）+ backend / model
suasor embeddings rebuild [--full] [--json] [--no-progress]  # 現行 model と異なる/欠落 source を再埋め込み（--full は全件）
suasor embeddings drain [--json] [--no-progress]  # pending（ベクトル未生成）の catch-up 再埋め込み
suasor embeddings list-failed [--limit N] [--json]  # 現行 model ベクトルを欠く source の drilldown（pending / stale）
suasor embeddings find-duplicates [--threshold T] [--json] [--no-progress]  # cosine 類似度が閾値超の near-dup ペア列挙
suasor search [--limit N] [--source-type T] [--observed-after ISO] [--observed-before ISO] [--json] <query>  # FTS 検索
suasor source list [--type T] [--limit N] [--since ISO] [--until ISO] [--json]  # 取り込み済み source の監査一覧（本文・secret 非表示・ADR-0026）
suasor source forget <externalId> [--reason R] [--yes]  # source をローカル purge（本文 redaction + projection 削除・破壊的・--yes で適用・ADR-0026）
suasor brief [--since D] [--until ISO] [--limit N] [--json]  # 期間ダイジェスト（brief バンドル）を stdout 出力
suasor mcp serve                       # MCP server（stdio）起動（read + HITL write tools）
suasor mcp tools [--json]              # MCP 登録ツールを server 起動せず一覧（name / read·write / 概要）
suasor slack auth set [--token T]      # Slack token を OS keychain に保存（省略時 stdin）
suasor slack auth test [--json]        # token 検証 + granted scopes + feature readiness
suasor slack conversations [--types T] [--include-archived] [--limit N] [--sort last_self_post] [--no-progress] [--json]  # 可視会話の列挙 + 設定ブロック出力
suasor slack status [--json]           # 保存中の resume cursor（workspace / channel）を表示（ts は人間可読列）
suasor slack cursor reset (--channel C1,C2 | --all) [--workspace A] [--yes]  # cursor を消し floor から取り直す
suasor slack cursor backfill --channel C1 --since 180d [--workspace A] [--yes]  # cursor を過去 floor へ下げ未取得分を取り直す
suasor slack resolve-names [--workspace A] [--force] [--no-progress] [--json]  # 既取り込み分の channel/user 名を遡及解決（ADR-0037）
suasor skills install [--scope S] [--host DIR] [--dry-run]  # アシスタント skill 展開
suasor skills list [--scope S] [--host DIR] [--format F] [--json]  # アシスタント skill 状態一覧（detailed で category/境界併記）
suasor skills search <kw> [--json]                          # name/description/category/triggers 横断検索
suasor skills info <name> [--json]                          # 単一 skill の詳細（category/境界/triggers/pairs/MCP tools）
suasor --version                       # バージョン出力
```

/ `search` / `source list` / `source forget`（取り込み済み source のローカル監査一覧 + 手動 purge・既存の listSources / sourceForget を CLI 公開・本文/secret 非表示・[ADR-0026](../adr/0026-source-forgetting.md)） / `brief`（期間バンドルを非対話に stdout 出力・定期実行向け・[ADR-0017](../adr/0017-brief-period-bundle.md)） / `<connector> sync` / `<connector> auth set` / `<connector> auth test`（github / ms-graph / google / box / notion / jira の汎用 auth verb・[ADR-0011](../adr/0011-slack-operational-verbs-and-readiness.md) を Slack 以外へ拡張）/ `github repos`（汎用 discovery verb 基盤の第 1 弾・token から可視リポジトリを列挙し paste-ready な `[connectors.github]` ブロックを出力・[ADR-0030](../adr/0030-connector-discovery-verbs.md)）/ `google calendars`（discovery verb 第 2 弾・refresh token から可視カレンダーを列挙し paste-ready な `[connectors.google]` `embeddings find-duplicates`（埋め込み層の保守 verb・[ADR-0006](../adr/0006-ml-delegation.md)）/ `mcp serve`（read + HITL write tools・[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md)）/ `mcp tools`（MCP tool surface introspection・[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md)）/ `slack auth set` / `slack auth test` / `slack conversations`（Slack 運用 verb・[ADR-0011](../adr/0011-slack-operational-verbs-and-readiness.md)）/ `slack status` / `slack cursor reset` / `slack cursor backfill`（cursor `<connector> sync` は connector registry から 1 connector = 1 command で派生する（[ADR-0007](../adr/0007-connector-contract.md)）。`sync`（全有効 connector の一括 one-shot 取り込み・[ADR-0027](../adr/0027-bulk-sync-orchestration.md)）/ `sync status`（connector 別の sync 鮮度＝最終 sync 時刻 / 件数 / 直近の成否を `sync_runs` projection から表示・[ADR-0033](../adr/0033-sync-run-history.md)）も稼働。
稼働 connector: `github` / `slack` / `ms-graph`（Outlook / Calendar / OneDrive / Teams）/ `google`（Drive / Gmail / Calendar）/ `box` / `web`（Playwright snapshot）/ `local`（ローカル FS 走査・[ADR-0023](../adr/0023-local-filesystem-connectors.md)）/ `notion`（Notion ページ / database）/ `jira`（issue / comment 取り込み）。setup は [connectors guide](../guide/connectors.md)。

## フラグ（確定）

| コマンド | フラグ | 既定 | 意味 |
|---|---|---|---|
| `init` | `--force` | false | 既存 `config.toml` を default テンプレートで上書きする |
| `onboard` | `--connector a,b` | (TTY 時は対話) | セットアップ対象の connector 名カンマ列。非 TTY（パイプ / CI）では必須 |

| `onboard` | `--json` | false | 各ステップ結果（auth / config 追記有無 / sync / scheduler 種別）を機械可読出力 |

- `config show` は実効 config（`env override > file > defaults` を合成した値・[config design](config.md)）を表示する値確認 verb。`doctor` が「何が wired / missing か」の **健全性診断**なのに対し、`config show` は「今どの値が効いているか」の **値そのもの**を出す（責務分離）。CI / Docker / headless で実効値を確認するのに使う。secret は**常にマスク**（NFR-PRV-4）: token は `config.toml` に保存されない（keychain / env override・`src/connectors/secrets.ts`）ため合成 config に secret 値は載らないが、万一 secret 風キーが混入しても `***` でマスクする防御を入れ、connector の資格情報は**存在有無のみ**（`set` / `unset`、値は読まない）を別ブロックで出す。`--effective`（既定）は合成値を出し、将来 `--source` 等で由来表示を足す余地を残す（`--no-effective` は今のところ代替挙動が無いため silent に無視せず error 終了する）。`--json` は `{config, credentials}` を出す。サービス本体は `src/cli/commands/config-show.ts`、masking ユーティリティは `src/config/mask.ts`
- `config edit` は既存 `config.toml` を `$EDITOR`（または `--editor <cmd>`）で開いて**保存後に schema 検証**する micro-edit verb（[Issue #280](https://github.com/ozzy-labs/suasor/issues/280)）。`onboard` が connector の**新規追加**、`config show` が**表示のみ**なのに対し、`config edit` は既存設定の手編集を担う。エディタ終了後に loader 同等の検証（`Bun.TOML.parse` + schema + strict connector slice・[ADR-0007](../adr/0007-connector-contract.md)）を走らせ、**不正な TOML / schema 違反なら元ファイルを復元して非ゼロ終了**するため壊れた config が残らない。write verb だが構造上 **HITL**（ユーザーがエディタで打鍵・自律適用なし・[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md)）。secret は書かない（token は keychain・config に載らない・NFR-PRV-4）。`config.toml` が無ければ `init` を案内して error。エディタ起動は注入可能（テスト用）。サービス本体は `src/cli/commands/config-edit.ts`
- `validate-config` は `config.toml` の**構造検証**verb（[Issue #280](https://github.com/ozzy-labs/suasor/issues/280)）。`doctor` が「環境が wired か」を診るのに対し、`validate-config` は「config ファイル自体が整合しているか」を診る。loader と同じ規則（schema + strict connector slice・[ADR-0007](../adr/0007-connector-contract.md)）で検証するが、最初の 1 件で fail-fast せず**全 finding を収集・分類**する: `missing-required`（必須欠落）/ `invalid-value`（型 / enum / range / regex 違反。不正な regex リテラルもここ）/ `unknown-key`（connector schema に無い typo・例 `repo` vs `repos`）/ `dangling-reference`（path 設定が存在しない先を指す・`[connectors.local].roots` の欠落 / `[storage].dbPath` の親 dir 欠落）。`--fix` は**除去のみの保守的修正**（unknown/typo キーの削除・dangling local root の削除）を**コメント / 整形を保つ surgical TOML 編集**で適用する。**値は捏造しない**ため `missing-required` / `invalid-value` は報告のみで自動修正しない（HITL・[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md)）。`[storage].dbPath` は `init` で後から作られうるので削除しない。**未修正 finding が残れば exit 1**（CI が gate に使える）。secret は読み書きしない（NFR-PRV-4）。サービス本体は `src/config/validate.ts`（検証）+ `src/config/toml-edit.ts`（surgical 編集）、CLI は `src/cli/commands/validate-config.ts`
- `export backup` は store dir の**整合バックアップ**を生成する read-only verb（[Issue #280](https://github.com/ozzy-labs/suasor/issues/280)）。local-first / event-sourced（event log が唯一の真実・[ADR-0002](../adr/0002-event-sourced-architecture.md)。projection / FTS / vec0 は replay で再構築可能）だが base install にバックアップ手段が無い点を解消する。スナップショットは SQLite `VACUUM INTO` で**読み取りロック下**に取得し WAL を畳み込むため、WAL/SHM サイドカーの分断（torn copy）を生まない。**無副作用**（live DB は一切変更しない）。secret は関与しない（token は keychain・DB に載らない・NFR-PRV-4）。`--format sqlite`（既定）は自己完結の単一 `.db`（正本・整合性チェックとして event 数を source と突き合わせる）、`--format tgz` は同スナップショットの gzip tar（live DB+WAL+SHM の 3 点ではなく vacuum 済み単一ファイルを tar するので checkpoint 状態に依らず常に整合）。`--out` 省略時は DB と同ディレクトリに timestamped 名で出力し、既存ファイルは上書き拒否。未 migrate / DB 不在は error。restore 手順は [data-audit guide](../guide/data-audit.md)。サービス本体は `src/db/backup.ts`、CLI は `src/cli/commands/export-backup.ts`
- `store info` は store の**規模**を可視化する read-only verb（[Issue #202](https://github.com/ozzy-labs/suasor/issues/202)）。`doctor` が「何が wired / missing か」を診断するのに対し、`store info` は event ログ件数（append-only・[ADR-0002](../adr/0002-event-sourced-architecture.md)）・projection table 11 種の行数・DB ファイルサイズ（WAL / SHM サイドカー込み）・vec0 ベクトル件数 + `embeddings_meta` 行数・FTS5 index 規模（[ADR-0005](../adr/0005-fts-first-retrieval-embedding-sidecar.md)）を出す。runaway な event log / 空の FTS index / 半端な vec0 を一望できる。COUNT / PRAGMA / stat のみで DB は作らない（未 migrate は error で `suasor init` / `db migrate` を案内）。vec0 / FTS table が無い store では当該カウントは `null`。`--breakdown` で event ログを type 別に集計（`COUNT(*) GROUP BY type`、read-only）して追加表示する（rebuild/replay デバッグや source 構成把握向け・[Issue #270](https://github.com/ozzy-labs/suasor/issues/270)）。`--json` は `StoreInfo`（`--breakdown` 併用時は `eventBreakdown` を追加）を出す。サービス本体は `src/db/store-info.ts`
- `source list` / `source forget` は取り込み済み source の**ローカル監査一覧**と**手動 purge（forget）**を CLI 公開する（[ADR-0026](../adr/0026-source-forgetting.md)・[data-audit guide](../guide/data-audit.md)）。従来 MCP（agent 経由）でのみ可能だった read query `listSources`（`src/mcp/queries.ts`）と forget ロジック `sourceForget`（`src/forget/source-forget.ts`）を、MCP クライアントなしで使えるよう薄く露出する verb。`source list` は read-only（自律 OK）で `observed_at` 降順に external id / `source_type` / `observed_at` のみを出し、**本文・secret は出さない**（NFR-PRV-4・`--json` も `{externalId, sourceType, observedAt}[]`）。`--type` / `--since` / `--until` / `--limit` で絞る。`source forget` は破壊的（event log 本文 redaction + projection/FTS/vector 削除 + body-less `SourceForgotten` 監査 event）なため `--yes` なしは preview のみで適用しない（[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md) HITL）。べき等で、forget 済み id は `already forgotten`・未取り込み id は `missing`（exit 1）。サービス本体は `src/cli/commands/source.ts`
- `connectors list` は connector registry（[ADR-0007](../adr/0007-connector-contract.md)、`<connector> sync` の派生元）を起動なしで introspect する。各 connector の `enabled`（`[connectors.<name>]` slice が存在し `enabled = false` でない）と token 設定有無（`resolveSecret` で env override → keychain を確認、**値は出さない**・NFR-PRV-4）を返す。auth 不要な connector（`web`）は token を `n/a`（JSON は `tokenConfigured: null`）。connector ごとの secret 名は registry の `SECRET_NAMES`（`src/connectors/registry.ts`）が SSOT。サービス本体は `src/cli/commands/connectors-list.ts`
- `embeddings status` / `embeddings rebuild` / `embeddings drain` / `embeddings list-failed` / `embeddings find-duplicates` は任意の埋め込み層（[ADR-0006](../adr/0006-ml-delegation.md)・[embedding setup](../guide/embedding.md)）を運用者から可視化・修復する保守 verb（#87・[Issue #202](https://github.com/ozzy-labs/suasor/issues/202)）。`status` は entity 種別（`sources.source_type`）ごとに embedded / pending（ベクトル未生成）/ stale（別 model で生成済み）を集計し、有効 backend / model を出す。`rebuild` は記録 model（`embeddings_meta` サイドカー）が現行 `[embedding].model`（+ version）と異なる/欠落の source を再埋め込み（`--full` は全件）、`drain` はベクトル未生成の pending のみ catch-up、`list-failed` は `status` の roll-up に対する drilldown で現行 model ベクトルを欠く source（`pending` / `stale`）を列挙し（pending 先頭・`--limit` 既定 50）、`find-duplicates` は vec0 のベクトル間 cosine 類似度が閾値超のペアを列挙する。いずれも `[embedding].backend` 無効時は明示メッセージで no-op 終了。`rebuild` / `drain` の埋め込みは best-effort でサイドカー失敗は warning（stderr）に留める（ingest と同じ degrade、[ADR-0006](../adr/0006-ml-delegation.md)）。ML はサイドカーへ委譲し本体は SQL + thin embedder client のみ（vec0 + `embeddings_meta` サイドカーは event ではない派生 substrate・[ADR-0002](../adr/0002-event-sourced-architecture.md)）。サービス本体は `src/retrieval/embedding/maintenance.ts`
- `mcp tools` は MCP tool surface（[ADR-0004](../adr/0004-mcp-agent-boundary-and-hitl.md)、[mcp-surface](mcp-surface.md)）を **server を起動せず**列挙する。各ツールの read/write 区分（`readOnlyHint`）と概要を出す。カタログは `src/mcp/tool-catalog.ts` をデータの SSOT とし、`tests/mcp/tool-catalog.test.ts` が実際に登録される server の tool（name / readOnlyHint）と突き合わせて drift を防ぐ。サービス本体は `src/cli/commands/mcp-tools.ts`
- `slack auth set` / `slack auth test` / `slack conversations` は Slack 固有の運用 verb（[ADR-0011](../adr/0011-slack-operational-verbs-and-readiness.md)）。汎用 connector 契約（`sync` のみ）は太らせず、token 保存（keychain）・scope 検証・会話 id 発見を担う。いずれも Slack SDK を読まず `fetch` のみ（import-clean）。`auth test` は `auth.test` 1 回で granted scopes（`x-oauth-scopes`）と feature readiness（`READY` / `READY (degraded)` / `MISSING <scope>` / `N/A`）を出す。readiness は scope 層のみで channel membership は保証しない。`conversations` は型ごとに `missing_scope` を自己申告し、`config.toml` に貼れる `[connectors.slack]` ブロックを出力する。サービス本体は `src/connectors/slack/`
- `slack conversations` の人間可読出力は各行を `Joined / ID / Name` の 3 列で並べる。先頭 `Joined` 列の `✓` は token の principal がその channel の member（= sync で到達可能）であることを示し、空欄は未参加を意味する。**`✓` の無い channel は `not_in_channel` を返し、bot が join / `/invite` されるまで ingest が 0 件**になる（[ADR-0011](../adr/0011-slack-operational-verbs-and-readiness.md)）。未参加 channel が 1 つでもある場合のみ、この含意を説明する note を stderr に出す（全 join 済みなら出さない）。`ID` 列が `channels` 設定に貼る値（名前ではなく id・#158）。`--json` 出力は列整形を行わず member 情報は raw のまま返す
- `skills install` は SSOT `docs/skills/<name>/SKILL.md`（パッケージ同梱）を `<host>/.claude/skills/<name>/SKILL.md` / `<host>/.agents/skills/<name>/SKILL.md` に展開する（[ADR-0008](../adr/0008-assistant-skills.md)）。冪等で、内容一致は `unchanged`・欠落は `created`・差分は SSOT で `updated`。エコシステム共通 dev skill（`@ozzylabs/skills`）は名前空間 disjoint で touch しない。サービス本体は `src/skills/`
- `skills list` は host dir ごとに各 skill を `installed`（SSOT と一致）/ `missing`（未展開）/ `modified`（展開済みだが SSOT と差分）で報告する。[ADR-0035](../adr/0035-project-skills-vendor-dev-skills.md) で in-repo dogfood-commit と `skills-drift` フックは廃止された。assistant mirror は `.gitignore` 済みのローカル install 物（commit しない）で、host dir に commit されるのは vendored dev skill（drive / lint 等）のみ。install 正しさは `tests/skills/install.test.ts` が担保する
- 長時間コマンド（`<connector> sync` / `sync` / `projections rebuild` / `embeddings rebuild` / `embeddings drain` / discovery verb 群 / `slack conversations` / `slack resolve-names`）の TTY 進捗表示は実装済み（`src/cli/progress.ts` の `createProgress`・stderr / TTY 限定・`--no-progress` で無効化）。env 上書きは現状未導入（必要が出れば別途検討）
